## How the file tree compares

A NestJS or Express project has its own conventions. FastAPI has none enforced by the framework вЂ” but the community has settled on a pattern that will feel familiar.

The mapping is almost 1:1:
- `controller` в†’ `routers/` (uses `APIRouter`)
- `service` в†’ you can add a `services/` layer вЂ” FastAPI does not enforce it
- `dto` в†’ `schemas/` (Pydantic models)
- `entity` в†’ `models/` (SQLAlchemy models)
- `app.module.ts` в†’ `main.py`

## main.py вЂ” the entry point

export const mainTs = `// Express src/main.ts
import express from 'express';
import { itemsRouter } from './items/items.router';

const app = express();
app.use(express.json());
app.use('/items', itemsRouter);

app.listen(3000, () => console.log('listening on 3000'));`;

export const mainPy = `# app/main.py
from fastapi import FastAPI
from app.routers import items

app = FastAPI(title="Items API", version="1.0.0")

app.include_router(items.router, prefix="/items", tags=["items"])

# Optional startup/shutdown events
@app.on_event("startup")
async def startup():
    print("Server is starting up")

# Run: uvicorn app.main:app --reload`;

<TsGo ts={mainTs} go={mainPy} />
