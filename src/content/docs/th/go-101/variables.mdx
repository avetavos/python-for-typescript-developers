## ตัวแปรใน TypeScript เทียบกับ Go

ใน TypeScript คุณมี `let` สำหรับตัวแปรที่เปลี่ยนค่าได้, `const` สำหรับการ bind ค่าที่เปลี่ยนไม่ได้ และ type inference ผ่าน `:` คุณยังคุ้นเคยกับค่าที่เริ่มต้นเป็น `undefined` ก่อนการกำหนดค่าด้วย

Go มีกลไกที่คล้ายกันแต่มีกฎที่สะอาดกว่า และมีแนวคิดหนึ่งที่ TypeScript ไม่มีคำตอบโดยตรง นั่นคือ **zero values**

## การประกาศตัวแปร

Go มีสองรูปแบบ การประกาศแบบสั้น `:=` (ใช้ได้เฉพาะภายในฟังก์ชัน) จะ infer type จากฝั่งขวามือ — เหมือนกับ `let x =` ใน TypeScript ส่วนรูปแบบยาว `var` ใช้ในระดับ package หรือเมื่อคุณต้องการประกาศโดยยังไม่กำหนดค่าทันที

export const tsVarCode = `// TypeScript
let count: number = 0;
let name = "Alice";   // inferred as string
const MAX = 100;`;

export const goVarCode = `// Go
var count int = 0
name := "Alice"   // inferred as string
const MAX = 100`;

<TsGo client:visible ts={tsVarCode} go={goVarCode} />

## Type inference ด้วย :=

โอเปอเรเตอร์ `:=` คือตัวหลักของ Go สำหรับตัวแปรในระดับ local มันทั้งประกาศและกำหนดค่าในขั้นตอนเดียว — ไม่ต้องมี `let` ไม่ต้องระบุ type annotation เมื่อ type ชัดเจนอยู่แล้ว

export const tsInferCode = `let score = 42;        // number
let ratio = 3.14;      // number
let label = "hello";   // string
let active = true;     // boolean`;

export const goInferCode = `score := 42          // int
ratio := 3.14        // float64
label := "hello"     // string
active := true       // bool`;

<TsGo client:visible ts={tsInferCode} go={goInferCode} />

## Constants

ค่าคงที่ใน Go ทำงานเหมือน `const` ใน TypeScript — เปลี่ยนแปลงไม่ได้ และถูกประเมินค่าตอน compile time Go ยังมี `iota` สำหรับสร้างลำดับแบบ enum ที่เพิ่มค่าอัตโนมัติ (จะกล่าวถึงภายหลัง)

export const tsConstCode = `const PI = 3.14159;
const APP_NAME = "MyApp";`;

export const goConstCode = `const PI = 3.14159
const AppName = "MyApp"`;

<TsGo client:visible ts={tsConstCode} go={goConstCode} />

<GoOnly title="Zero Values — ไม่มี undefined ใน Go">

ใน TypeScript ตัวแปรที่ประกาศแต่ยังไม่กำหนดค่าจะเป็น `undefined` แต่ใน Go ทุก type มี **zero value** — ค่าเริ่มต้นที่ได้รับเมื่อประกาศโดยไม่กำหนดค่า ไม่มี `undefined` ไม่มี `null` สำหรับ value types

| Go type | Zero value |
|---------|-----------|
| `int`, `float64` | `0`, `0.0` |
| `string` | `""` (string ว่าง) |
| `bool` | `false` |
| pointer, slice, map, channel, func | `nil` |

```go
var x int     // x == 0
var s string  // s == ""
var b bool    // b == false
```

สิ่งนี้ทำให้โปรแกรม Go คาดเดาได้ — คุณจะไม่เผลอไปทำงานกับ `undefined` โดยไม่ตั้งใจ

</GoOnly>
