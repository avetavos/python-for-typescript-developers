## Jest `it.each` vs Go table-driven tests

ใน Jest คุณใช้ `it.each` หรือ `describe.each` เมื่อต้องการรัน assertion logic เดิมซ้ำกับ inputs หลายอัน โดยไม่ต้องคัดลอก test functions Go ไม่มี `it.each` ในตัว — แต่ community ตกลงกัน idiom สากลแทน: slice ของ anonymous structs ("table") ที่วนลูปใน test function เดียว โดยแต่ละ case รันเป็น **subtest** ผ่าน `t.Run`

> หมายเหตุเกี่ยวกับ Playground: `testing` package ต้องการ binary context ของ `go test` บล็อกโค้ดด้านล่างแสดงไฟล์ test จริง — รัน locally ด้วย `go test ./...`

## เปรียบเทียบ: ทดสอบฟังก์ชันเดิม

export const tsTableCode = `// Jest — it.each
function add(a: number, b: number) { return a + b; }

it.each([
  [1,  2,  3],
  [0,  0,  0],
  [-1, 1,  0],
  [10, 20, 30],
])('add(%i, %i) = %i', (a, b, want) => {
  expect(add(a, b)).toBe(want);
});`;

export const goTableCode = `// Go — table-driven ด้วย t.Run
func add(a, b int) int { return a + b }

func TestAdd(t *testing.T) {
    tests := []struct {
        name string
        a, b int
        want int
    }{
        {"positive", 1, 2, 3},
        {"zeros",    0, 0, 0},
        {"negative", -1, 1, 0},
        {"large",    10, 20, 30},
    }

    for _, tc := range tests {
        t.Run(tc.name, func(t *testing.T) {
            got := add(tc.a, tc.b)
            if got != tc.want {
                t.Errorf("add(%d, %d) = %d; want %d", tc.a, tc.b, got, tc.want)
            }
        })
    }
}`;

## กายวิภาคของ table-driven test

```go
// math_test.go
package math_test

import "testing"

func TestDivide(t *testing.T) {
    // 1. Table — slice ของ anonymous structs
    tests := []struct {
        name    string
        a, b    float64
        want    float64
        wantErr bool
    }{
        {"normal",         10, 2, 5,   false},
        {"fraction",        7, 2, 3.5, false},
        {"divide_by_zero",  1, 0, 0,   true},
    }

    // 2. วนลูปใน table
    for _, tc := range tests {
        // 3. t.Run สร้าง named subtest
        t.Run(tc.name, func(t *testing.T) {
            // 4. t.Parallel() — optional, รัน subtests พร้อมกัน
            t.Parallel()

            got, err := Divide(tc.a, tc.b)

            if (err != nil) != tc.wantErr {
                t.Fatalf("Divide() error = %v, wantErr %v", err, tc.wantErr)
            }
            if !tc.wantErr && got != tc.want {
                t.Errorf("Divide() = %v; want %v", got, tc.want)
            }
        })
    }
}
```

conventions หลัก:
- `t.Errorf` mark ว่า fail แต่ยังคง continue test
- `t.Fatalf` mark ว่า fail และหยุด subtest ปัจจุบันทันที
- `t.Parallel()` ภายใน `t.Run` อนุญาตให้ subtests รันพร้อมกัน (ปลอดภัยเมื่อ cases เป็นอิสระ)
- field `name` ในแต่ละ case กลายเป็น subtest label (`TestDivide/normal`, `TestDivide/divide_by_zero`)

## รัน specific subtests

```bash
# รัน tests ทั้งหมด
go test ./...

# รันเฉพาะ subtest ที่มีชื่อ (รองรับ regex)
go test -run TestDivide/divide_by_zero ./...

# รัน subtests ทั้งหมดที่ชื่อมี "zero"
go test -run TestDivide/.*zero.* ./...

# Verbose output — แสดงชื่อ subtest แต่ละตัวและ PASS/FAIL
go test -v ./...
```

## เปรียบเทียบโครงสร้างกับ Jest

export const jestStructureCode = `// Jest
describe('Divide', () => {
  it.each([
    ['normal',         10, 2, 5],
    ['fraction',        7, 2, 3.5],
  ])('%s', (_, a, b, want) => {
    expect(divide(a, b)).toBe(want);
  });

  it('throws on divide by zero', () => {
    expect(() => divide(1, 0)).toThrow();
  });
});`;

export const goStructureCode = `// Go
func TestDivide(t *testing.T) {
    tests := []struct {
        name    string
        a, b    float64
        want    float64
        wantErr bool
    }{
        {"normal",   10, 2, 5,   false},
        {"fraction",  7, 2, 3.5, false},
        {"by_zero",   1, 0, 0,   true},
    }
    for _, tc := range tests {
        t.Run(tc.name, func(t *testing.T) {
            got, err := Divide(tc.a, tc.b)
            if (err != nil) != tc.wantErr { t.Fatal(...) }
            if !tc.wantErr && got != tc.want { t.Errorf(...) }
        })
    }
}`;

<GoOnly title="Testify — ไลบรารี assertion ที่เป็น optional">

`testing` package มาตรฐานของ Go ไม่มี DSL แบบ `expect(...).toBe(...)` หากคุณพบว่า `t.Errorf` verbose เกินไป community มักใช้ [`github.com/stretchr/testify/assert`](https://github.com/stretchr/testify):

```go
import "github.com/stretchr/testify/assert"

func TestAdd(t *testing.T) {
    assert.Equal(t, 3, add(1, 2))
    assert.NoError(t, err)
    assert.EqualError(t, err, "division by zero")
}
```

Testify เป็น optional — standard library เพียงพอสำหรับ projects ส่วนใหญ่ table-driven pattern ทำงานเหมือนกันทั้งกับและไม่มีมัน

</GoOnly>
