## คุณคิดเป็น type อยู่แล้ว — Go ตอบแทนความสามารถนั้น

ถ้าคุณเขียน TypeScript อยู่ตอนนี้ คุณเข้าใจคุณค่าของ static type, compiler error, และ interface ที่ออกแบบมาอย่างจงใจอยู่แล้ว Go พาแนวคิดนั้นไปอีกขั้น: ไม่มีการลบ type ตอน runtime, ไม่มีขั้นตอนคอมไพล์แยกที่ "ข้ามได้" ในระหว่าง develop, และไม่มีโฟลเดอร์ `node_modules` ที่เต็มไปด้วย dependency หลายพันตัว

คอร์สนี้สมมติว่าคุณคุ้นเคยกับ TypeScript และ Node.js อยู่แล้ว ทุกแนวคิดจะอธิบายผ่านการเปรียบเทียบ — สิ่งที่คุณรู้แล้ว กับวิธีที่ Go ทำสิ่งเดียวกัน (หรือเหตุผลที่ Go ทำต่างออกไป)

## ทำไมต้องเรียน Go?

### คอมไพล์เป็น binary ตัวเดียว

TypeScript คอมไพล์เป็น JavaScript ซึ่งยังต้องการ Node.js runtime เพื่อรัน Go คอมไพล์ตรงไปเป็น native machine code binary คุณส่งไฟล์เดียว ไม่ต้องการ runtime บน target machine

export const tsDistCode = `// ส่ง: node dist/index.js
// ต้องการ: Node.js runtime ติดตั้งไว้แล้ว
// ขนาด: node_modules/ อาจถึง 200+ MB`;

export const goDistCode = `// ส่ง: ./myapp (binary เดียว, ~5 MB)
// ต้องการ: ไม่มีอะไรเพิ่มบน server
// $ go build -o myapp .`;

### Static type — ไม่มีเซอร์ไพรส์ตอน runtime

Type ของ TypeScript มีอยู่แค่ตอน development เท่านั้น และถูกลบออกก่อนที่โค้ดจะรัน Type ของ Go ถูก enforce โดย compiler และยังคงอยู่ตอน runtime โปรแกรมที่คอมไพล์ผ่านใน Go นั้น type-safe ตลอดการทำงาน

### เริ่มต้นเร็วและ throughput สูง

Node.js (V8) เร็ว แต่มี JIT warm-up time และ garbage collector ที่ปรับแต่งสำหรับ server ที่รันนานๆ Go เริ่มต้นใน millisecond, จัดการ concurrent connection หลายแสนรายการผ่าน goroutine และ GC มี pause ต่ำกว่า millisecond สำหรับ CLI, microservice, และเครื่องมือระดับ system ประสิทธิภาพของ Go เป็นเรื่องยากที่จะเอาชนะ

### Concurrency ระดับ first-class ด้วย goroutine

Node.js ทำงานแบบ single-threaded และใช้ event loop สำหรับ async I/O Go ใช้ goroutine — lightweight thread ที่ Go runtime จัดการแบบ multiplex บน OS thread จริงๆ คุณเขียนโค้ดที่ดูเหมือน sequential และ runtime จัดการ parallelism ให้ เรา preview สิ่งนี้ใน mental model lesson และครอบคลุมเชิงลึกในบทต่อๆ ไป

export const tsConcurrentCode = `// TypeScript: async/await + event loop
async function fetchAll(urls: string[]) {
  const results = await Promise.all(
    urls.map(url => fetch(url).then(r => r.json()))
  );
  return results;
}`;

export const goConcurrentCode = `// Go: goroutine + channel
func fetchAll(urls []string) []any {
    results := make([]any, len(urls))
    var wg sync.WaitGroup
    for i, url := range urls {
        wg.Add(1)
        go func(i int, url string) {
            defer wg.Done()
            // fetch logic here
            _ = i; _ = url
        }(i, url)
    }
    wg.Wait()
    return results
}`;

### Error แบบ explicit แทน exception

TypeScript โยน exception คุณครอบโค้ดด้วย `try/catch` และหวังว่าจะไม่มีอะไรหลุดรอด Go คืน error เป็นค่าธรรมดา ทุก function ที่อาจล้มเหลวจะบอกใน return type และคุณจัดการ error ทันทีที่ call site — ไม่มีอะไรให้ "ลืม"

export const tsErrorCode = `// TypeScript: exception สามารถ bubble ได้อย่างเงียบๆ
try {
  const data = JSON.parse(input);
  processData(data);
} catch (err) {
  console.error("something went wrong", err);
}`;

export const goErrorCode = `// Go: error คือ return value, จัดการแบบ explicit
data, err := parseInput(input)
if err != nil {
    return fmt.Errorf("failed to parse input: %w", err)
}
processData(data)`;

## คอร์สนี้ครอบคลุมอะไรบ้าง

คอร์สนี้พาคุณจากความรู้ Go เป็นศูนย์ไปสู่การเขียน Go แบบ idiomatic พร้อม production โดยใช้ประสบการณ์ TypeScript ของคุณเป็นสะพาน นี่คือสิ่งที่เราครอบคลุม:

| Module | หัวข้อ |
|--------|--------|
| **Intro** (module นี้) | ทำไมต้อง Go, การเปลี่ยน mindset, setup, hello world, layout ของโปรเจกต์ |
| **Go 101** | Variables, functions, control flow, errors, structs, interfaces, collections, packages |
| *(เร็วๆ นี้)* | Goroutines, channels, HTTP servers, databases, testing |
