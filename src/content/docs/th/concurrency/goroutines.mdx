## Async Functions vs Goroutines

ใน TypeScript เมื่อคุณต้องการรันโค้ดแบบ asynchronous คุณใช้ `async/await`:

```typescript
async function processUser(id: number) {
  const user = await fetchUser(id);
  return user;
}
// ต้อง await หรือ .then() เพื่อรอผล
await processUser(1);
```

ใน Go คุณเพิ่ม keyword `go` ด้านหน้า function call ใดก็ได้เพื่อรันมันใน goroutine แยกต่างหาก:

```go
go processUser(1) // รันในพื้นหลังทันที — ไม่ต้อง async keyword
```

ความแตกต่างสำคัญ: JavaScript `async` function ยังคงรันบน thread เดียว แต่ goroutine รันบน thread จริงที่ Go runtime จัดการ

## การสร้าง Goroutines

export const tsAsyncCode = `// TypeScript: async/await (single-threaded)
async function greet(name: string): Promise<void> {
  console.log(\`Hello from \${name}\`);
}

// ต้อง await เพื่อรอผล
await greet("Alice");
await greet("Bob");   // รันตามลำดับ

// Promise.all เพื่อรันพร้อมกัน (แต่ยังคง single-threaded)
await Promise.all([greet("Alice"), greet("Bob")]);`;

export const goGoroutineCode = `// Go: goroutines (truly concurrent)
package main

import (
    "fmt"
    "sync"
)

func greet(name string, wg *sync.WaitGroup) {
    defer wg.Done() // แจ้ง WaitGroup เมื่อเสร็จ
    fmt.Printf("Hello from %s\\n", name)
}

func main() {
    var wg sync.WaitGroup
    names := []string{"Alice", "Bob", "Carol"}

    for _, name := range names {
        wg.Add(1)         // ลงทะเบียนก่อน launch
        go greet(name, &wg) // launch goroutine
    }

    wg.Wait() // block จนกว่าทุก goroutine เสร็จ
    fmt.Println("All done!")
}`;

## sync.WaitGroup — รอ Goroutines ให้เสร็จ

`sync.WaitGroup` เป็นวิธีหลักในการรอ goroutines ทำงานเสร็จ ทำงานเหมือน `Promise.all` แต่ใช้ counter:

1. `wg.Add(n)` — เพิ่ม counter ก่อน launch goroutine
2. `wg.Done()` — ลด counter เมื่อ goroutine เสร็จ (มักใช้กับ `defer`)
3. `wg.Wait()` — block จนกว่า counter เป็น 0

> **สำคัญ:** ต้อง `wg.Add(1)` ก่อน `go f()` เสมอ ไม่ใช่ข้างใน goroutine เพราะ goroutine อาจยังไม่ได้รันตอนที่ WaitGroup ถูก Wait

## Anonymous Goroutines

เหมือน arrow functions ใน JS คุณสามารถ launch goroutine แบบ anonymous ได้:

export const tsAnonCode = `// TypeScript: immediately invoked async
(async () => {
  const result = await doSomething();
  console.log(result);
})();`;

export const goAnonCode = `// Go: immediately launched goroutine
go func() {
    result := doSomething()
    fmt.Println(result)
}() // วงเล็บ () ด้านหลังเพื่อ invoke ทันที`;

OS thread ปกติใช้ stack ประมาณ 1–8MB และมี overhead จาก kernel context switching

Goroutine ต่างกันอย่างสิ้นเชิง:
- Stack เริ่มต้นเพียง **2KB** และขยายได้ dynamic ตามต้องการ
- Scheduled โดย **Go runtime scheduler** (M:N threading) — ไม่ใช่ kernel โดยตรง
- สร้างได้หลักแสนถึงล้านตัวในแอปเดียว

export const playgroundGoroutinesCode = `package main

import (
	"fmt"
	"sync"
)

func printSquare(n int, wg *sync.WaitGroup) {
	defer wg.Done()
	fmt.Printf("%d² = %d\\n", n, n*n)
}

func main() {
	var wg sync.WaitGroup
	for i := 1; i <= 5; i++ {
		wg.Add(1)
		go printSquare(i, &wg)
	}
	wg.Wait()
	fmt.Println("All squares printed")
}`;
